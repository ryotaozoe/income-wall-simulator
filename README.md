# 年収の壁シミュレーター（2025年税制改正対応）

学生・アルバイト向けに、**年収の見込みから「親の扶養への影響」と「手取り額」を自動計算**する Web アプリケーションです。
2025年（令和7年度）の税制改正で複雑化した「年収の壁」を、正確な条件分岐とテストで実装しました。

> お金に関わるシステムは **計算の正確性とバグのなさ** が命です。
> そのため型の厳密な **Java（Spring Boot）** を採用し、税制ロジックを単体テストで担保しています。

---

## 🎯 背景・解決したい課題

「アルバイトをいくらまで稼いでいいか分からない」という学生の悩みを解決します。

2025年の税制改正で「年収の壁」は大きく変わり、特に **19〜22歳には「特定親族特別控除」が新設**されました。
これにより従来の「150万円を超えると控除が消える崖」が、**188万円までの段階的な減少（スロープ）**に変わるなど、
制度が複雑化しています。本アプリは、この複雑な最新ルールを正確に計算し、
**「あといくら稼げるか」「扶養を外れると親の負担がいくら増えるか」**を可視化します。

---

## ✨ 主な機能

- **年収の壁の判定**：106万 / 130万（社会保険）、160万（所得税）、150万・188万（特定親族特別控除）の超過をリアルタイム判定
- **手取り額の概算**：所得税・住民税・社会保険料を差し引いた手取りを算出
- **親への影響の可視化**：扶養控除の減少額と、親の追加税負担（概算）を表示
- **条件に応じた分岐**：年齢区分（19〜22歳か）・勤務先の規模・親の就業形態でロジックが変化
- **シミュレーション履歴**：過去の試算結果を MySQL に保存して一覧表示

## 💡 設計・実装のこだわり

### 1. 型の厳密な Java による正確な計算
お金の計算でバグは許されません。整数型による厳密な計算と、入力値の `@Valid` バリデーション（0〜1000万円）で不正値を防いでいます。

### 2. 2025年税制改正の正確な反映
公式・税理士監修の一次情報を調査し、以下を正確に実装しました。

| 項目 | 2025年改正後の内容 |
|------|------|
| 基礎控除 | 合計所得に応じて 58〜95万円（学生帯は **95万円**・時限措置） |
| 給与所得控除 | 最低保障 **65万円**（給与収入190万円以下は一律） |
| 本人の所得税の壁 | **160万円**（基礎控除95万＋給与所得控除65万） |
| 特定親族特別控除（19〜22歳） | 123万円超でも **188万円まで段階的**に控除が残る |
| 社会保険の被扶養（19〜22歳） | 2025年10月から年収要件が **130万→150万円未満**に引き上げ（106万の壁は存続） |

特に「特定親族特別控除」は、給与収入に応じて控除額が段階的に減少するテーブルとして実装しています
（150万→63万、155万→61万、…、188万→3万、188万超→0万）。

### 3. 単体テストによる品質担保
税制ロジック（`TaxCalculationService`）に対し、**各「壁」の境界値**と**段階的控除**を中心に
**25件の単体テスト**を実装しています。`@Nested` / `@ParameterizedTest` で構造的に網羅しました。

テストは本番の MySQL ではなく **H2 インメモリ DB** を使用するため、DB 環境がなくてもどこでも実行できます。

### 4. 関心の分離（レイヤードアーキテクチャ）
```
Controller（リクエスト受付・バリデーション）
  └─ Service（税制計算ロジック）
       └─ Repository（永続化）
            └─ Entity / DTO（データ構造）
```
計算ロジックを `Service` に集約し、Web や DB から独立してテスト可能にしています。

## 🚀 セットアップと起動

### 前提
- Java 21
- MySQL 9（起動済み）

### 1. データベースの作成
```bash
mysql -u root -e "CREATE DATABASE income_wall_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

### 2. アプリの起動
```bash
./mvnw spring-boot:run
```
ブラウザで [http://localhost:8080](http://localhost:8080) にアクセスします。

### 3. テストの実行
```bash
./mvnw test
```
> テスト時は H2 インメモリ DB を使用するため、MySQL の起動は不要です。
